# Промпты исследовательского трио гипотезы конструкта

## Статус

Инфраструктура исследовательского контура. Не является нормативным слоем корпуса.

Промпты задают режимы двух LLM внутри **трио**. Автор гипотезы и оператор фиксации результатов — человек-исследователь. Источник истины по определениям — словарь и тома.

Мотивация, динамика уточнения и посадка на гипотезу корпуса — в `notes/010_Исследовательское_трио_как_режим_II_порядка.md` (для последующих переоценок контура).

---

# Суть трио: конкуренция второго порядка

Трио — не схема «генерация → veto → тишина».

Это организация, в которой конкурируют **информационные структуры** (предложения, ограничения, контрпримеры), а не носители и не право голоса. Победа одной структуры не уничтожает остальные. Любой завершённый цикл **обязательно** оставляет сохранённый результат в общем пространстве познания трио.

Это прямая корреляция с конкуренцией второго порядка и принципами неубывающей плотности / кумулятивной эволюции (словарь, том II, `notes/004`).

## Инвариант выхода

| Исход цикла | Что уплотняется | Аналог в корпусе |
| --- | --- | --- |
| Рабочая заметка | пространство эвристик, рифм, напряжений | `notes/` |
| Усиление гипотезы | связность и выразимость ядра | словарь / том / явный импорт |
| Опровержение предположения | пространство известных ограничений | антиподмена, отрицательный пример, запись в evolution |

«Опровержение» здесь не уничтожение ветви. Отвергнутое предположение **сохраняется** как ограничение последующего поиска: стало известно, что этот ход ломает референт / недостаточен / преждевременен.

Пустой выход («просто отклонили и забыли») — режим шума или конкуренции первого порядка. Для трио он недопустим.

Следствие: при фиксации результатов суммарная плотность пространства познания трио не убывает. Отсюда рабочая формула: **результат цикла = 100%** в одном из трёх видов (или в их комбинации по тезисам), а не «иногда полезно, иногда в корзину».

# Роли

```text
                         исследователь гипотезы
                    (траектория модели + фиксация памяти)
                              /            \
                             /              \
                            v                v
              [1] исследователь границ    [2] корневой аудитор
                   структура A                структура B
              расширение кандидатов      проверка границ / референтов
                            \              /
                             \            /
                              v          v
                    сохранённый исход второго порядка:
                 заметка | усиление ядра | опровержение-как-ограничение
```

| Роль | Кто | Функция в трио |
| --- | --- | --- |
| Исследователь гипотезы | человек | держит императив и траекторию модели; запускает цикл; **гарантирует фиксацию** одного из трёх исходов; необратимо меняет пространство последующего поиска корпуса |
| Исследователь границ | LLM + `01` | производит конкурирующие структуры-кандидаты (нарушения, контрпримеры, рифмы) |
| Корневой аудитор | LLM + `02` | производит конкурирующие структуры-ограничения (подмена, ложная достаточность, преждевременное закрытие) |

Два LLM — не судьи друг над другом и не источники veto. Они порождают структуры, чья конкуренция имеет смысл только если исследователь **сохраняет** итог в памяти корпуса трио.

# Чем это не является

| Ложная рамка | Почему ломает суть |
| --- | --- |
| Корневой аудитор накладывает veto | Победившая ветвь уничтожает проигравшую → логика первого порядка |
| Исследователь границ создаёт шум на выходе | Шум = структуры без сохранения в пространстве поиска |
| «Отклонено» = конец без записи | Теряется вклад отвергнутого решения в плотность |
| Согласие двух LLM = импорт | Подмена фиксации исследователем автоматическим консенсусом |
| Расхождение LLM = сбой | Расхождение — нормальный материал конкуренции структур |

# Конвейер цикла

1. Исследователь задаёт вопрос / вход (текст, файлы, тезис).
2. При необходимости — `01` (структуры-кандидаты).
3. `02` по входу или по выводу `01` (структуры-ограничения и классификация).
4. Исследователь **обязательно** фиксирует исход:
   - рабочая заметка; и/или
   - усиление гипотезы (подготовка/внесение в нормативный слой); и/или
   - опровержение предположения как явное ограничение (антиподмена, запись в `HYPOTHESIS_EVOLUTION` / заметку / changelog).

Шаг 4 — не «решение, выкинуть ли». Это акт кумулятивной эволюции трио.

Допустим порядок `02` → точечный `01` по одному напряжению. Инвариант выхода тот же.

# Вердикты корневого аудитора (язык второго порядка)

| Вердикт | Не значит | Значит для фиксации |
| --- | --- | --- |
| `notes` | «второсортный мусор» | кандидат рабочей заметки |
| `strengthen` | «уже в ядре» | кандидат усиления гипотезы (бывш. import-candidate) |
| `refute-keep` | «уничтожить и забыть» | опровержение/ограничение сохранить в пространстве поиска |

Слово `reject` в этом контуре не используется: оно тянет семантику первого порядка.

# Ограничения

- Трио уплотняет пространство познания только при фиксации исходов; чат без записи — риск режима шума.
- Промпты сами не закрывают открытые вопросы `ROADMAP`.
- Новый нормативный термин по-прежнему сначала в словарь.
- Трио не отменяет различие `notes/` и ядра: оба пути — валидные исходы второго порядка, разные по слою корпуса.
