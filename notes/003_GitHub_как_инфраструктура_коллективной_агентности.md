# GitHub как инфраструктура коллективной агентности

## Статус

Исследовательская заметка.

---

# Тезис

GitHub может рассматриваться как инфраструктура коллективной агентности, но не как самостоятельный агент.

Его значение для гипотезы конструкта состоит в том, что он делает видимой конкуренцию второго порядка и кумулятивную информационную эволюцию.

---

# Механизмы

GitHub поддерживает:

- память изменений через историю коммитов;
- ветвление через branches;
- проверку через review и обсуждения;
- отбор через pull request'ы;
- преемственность через сохранение репозитория независимо от отдельных участников;
- повторное использование через forks и dependencies.

---

# Конкуренция второго порядка

Представим два проекта.

Проект A оказался лучше.

Проект B закрыли.

Произошло не это:

A + B → A

А скорее:

A₀

\

+ идеи B

/

A₁

Даже если код проекта B никогда больше не используется, остаются:

- архитектурные решения;
- найденные ошибки;
- опыт разработчиков;
- обсуждения;
- документация;
- новые компетенции участников.

Информация не исчезает. Она перераспределяется и часто интегрируется в последующие структуры.

---

# Связь с гипотезой конструкта

В терминах гипотезы GitHub-подобная система позволяет наблюдать:

- рост информационной плотности;
- конкуренцию второго порядка;
- принцип неубывающей информационной плотности;
- принцип кумулятивной информационной эволюции;
- иерархическую компрессию через повторно используемые абстракции;
- парадокс информационной плотности и его преодоление;
- сохранение отвергнутых решений;
- переход локальных изменений в коллективную структуру;
- возникновение новых строительных блоков.

GitHub — не просто хранилище кода. Это механизм создания иерархии повторно используемых абстракций.

Коллективный агент здесь определяется не числом участников, а способностью системы сохранять результаты собственного поиска так, что они необратимо изменяют пространство последующего поиска.
